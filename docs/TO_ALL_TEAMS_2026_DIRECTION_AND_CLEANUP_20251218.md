## 1. Confluence 문서 검증 요청

### 배경
Confluence 페이지 ([01. 2026](https://ktspace.atlassian.net/wiki/spaces/CNCORE/pages/548700691)) 하위에 3개 팀의 문서들이 수동으로 모아져 있습니다.

### 요청 사항
**각 팀은 본인 팀이 작성한 Confluence 문서가 현재 최신 버전과 정확히 일치하는지 디테일하게 검증해주세요.**

#### 검증 체크리스트
```
[ ] 1. 문서 제목/구조가 최신 상태인가?
[ ] 2. API 스펙이 현재 코드와 일치하는가?
[ ] 3. 설정 값(포트, URL, 환경변수)이 정확한가?
[ ] 4. 예제 코드가 동작하는가?
[ ] 5. 스크린샷/다이어그램이 현재 UI와 일치하는가?
[ ] 6. 버전 정보가 정확한가?
```

### 검증 결과 보고
각 팀은 검증 후 아래 형식으로 응답 문서를 작성해주세요:
```

```

**2. 히스토리화 (archive/ 폴더로 이동)**
- TO_팀명_* 형식의 요청/응답 문서

**3. 삭제**
- 중복 문서
- 임시 작성 후 버려진 문서
- 의미 없는 메모성 문서

## 3. 2026년 방향: 개발자 거버넌스 & 가이드

### 핵심 방향
> **"플랫폼 완성보다, 개발자들이 쉽게 연동할 수 있는 거버넌스/가이드에 집중"**

### 2026년 주요 목표

| 목표 | 설명 | 담당 |
|------|------|------|
| **Agent 개발 가이드** | 새로운 Agent를 K-Jarvis에 연동하는 완벽한 가이드 | Orchestrator + Agent Team |
| **MCP Server 개발 가이드** | 새로운 MCP Server를 K-ARC에 연동하는 완벽한 가이드 | K-ARC Team |
| **SDK 고도화** | k-jarvis-utils, k-arc-utils 안정화 및 문서화 | 각 팀 |
| **템플릿 제공** | Agent/MCP Server 개발 템플릿 및 보일러플레이트 | 전체 |
| **Confluence 가이드** | 개발자 온보딩 문서 완성 | 전체 |

### 우선순위
1. **P0**: Agent/MCP Server 연동 가이드 완성
2. **P1**: SDK 문서화 및 예제 추가
3. **P2**: 템플릿/보일러플레이트 제공
4. **P3**: 자동화 도구 (CLI, Builder)

## 4. 📌 향후 검토 사항: AgentHub 통합 (현재 미진행)

> ⚠️ **중요**: 이 섹션은 **향후 검토를 위한 인지 사항**입니다.  
> **현재는 진행하지 않습니다.** 다른 우선순위 작업에 집중해주세요.  
> 혼동을 피하기 위해 내용만 공유드리며, 추후 별도 킥오프 시 논의 예정입니다.

### 회사 요구사항 (참고용)
> **"Orchestrator의 Agent 카탈로그 관리 기능을 MCPHub(K-ARC)에 통합하여 'AgentHub'로 만들어 달라"**

### 현재 구조
```
┌─────────────────────┐     ┌─────────────────────┐
│   K-Jarvis          │     │   K-ARC (MCPHub)    │
│   Orchestrator      │     │                     │
│ ┌─────────────────┐ │     │ ┌─────────────────┐ │
│ │ Agent Catalog   │ │     │ │ MCP Server      │ │
│ │ Management      │ │     │ │ Management      │ │
│ └─────────────────┘ │     │ └─────────────────┘ │
│ ┌─────────────────┐ │     │                     │
│ │ Agent Routing   │ │     │                     │
│ │ & Orchestration │ │     │                     │
│ └─────────────────┘ │     │                     │
└─────────────────────┘     └─────────────────────┘
```

### 요구되는 변경
```
┌─────────────────────┐     ┌─────────────────────────────┐
│   K-Jarvis          │     │   K-ARC (AgentHub)          │
│   Orchestrator      │     │                             │
│                     │     │ ┌─────────────────────────┐ │
│ ┌─────────────────┐ │     │ │ Agent Catalog           │ │
│ │ Agent Routing   │◄├─────┤►│ Management (NEW)        │ │
│ │ & Orchestration │ │     │ └─────────────────────────┘ │
│ └─────────────────┘ │     │ ┌─────────────────────────┐ │
│                     │     │ │ MCP Server Management   │ │
│                     │     │ └─────────────────────────┘ │
└─────────────────────┘     └─────────────────────────────┘
```

### 논의 필요 사항

#### 1. 기능 분리 범위
| 기능 | 현재 위치 | 이전 여부 | 비고 |
|------|----------|----------|------|
| Agent 등록/삭제 | Orchestrator | ✅ 이전 | K-ARC로 |
| Agent 목록 조회 | Orchestrator | ✅ 이전 | K-ARC로 |
| Agent Health Check | Orchestrator | ❓ 논의 필요 | |
| Agent Card 관리 | Orchestrator | ✅ 이전 | K-ARC로 |
| Agent 라우팅 | Orchestrator | ❌ 유지 | Orchestrator 핵심 |
| Agent 호출/응답 | Orchestrator | ❌ 유지 | Orchestrator 핵심 |

#### 2. API 변경
- Orchestrator → K-ARC API 호출로 Agent 목록 조회
- Agent 등록 시 K-ARC에 등록, Orchestrator는 조회만

#### 3. DB 변경
- Agent 관련 테이블 K-ARC로 이전?
- 또는 K-ARC가 새로 관리?

#### 4. UI 변경
- K-Jarvis Frontend의 Agent 관리 UI → K-ARC로 이동?
- 또는 K-ARC에서 통합 관리 UI 제공?

### 팀별 역할 (참고용 - 추후 논의 시 검토)

| 팀 | 예상 역할 |
|----|----------|
| **Orchestrator Team** | Agent 라우팅/오케스트레이션 기능 유지, Agent 조회 API 변경 검토 |
| **K-ARC Team** | Agent Catalog 관리 기능 수용 가능 여부 검토 |
| **Agent Team** | Agent 등록 방식 변경 영향도 검토 |

> 📅 **별도 킥오프 미팅 시 상세 논의 예정** (일정 미정)
